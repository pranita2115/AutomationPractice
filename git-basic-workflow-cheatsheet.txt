*********** Git Basic Daily Workflow ***************

PART 1: Safe Workflow (with git status checks)
Use this when you are learning or want to verify each step.

1. git status

2. git add <file-name>

3. git status

4. git commit -m "add/update/remove: <short action message>"

5. git status

6. git push

7. git status


8. Final Step:
- Go to your Git server -GitHub
- Refresh the page
- Verify changes (added / deleted / updated files)


PART 2: Fast Workflow (without repeated checks)

Use this in real projects for speed.

1. git add <file-name>

2. git commit -m "add/update/remove: <short action message>"

3. git push

4. Final Step:

- Open your Git server
- Refresh repository
- Confirm changes are reflected

Quick Notes
- git add → stages changes
- git commit → saves snapshot locally
- git push → sends changes to remote repo
- git status → shows current state of repo
